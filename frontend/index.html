02

How it works

One atomic transaction
1
Route

The router scans every pool for the pair and picks the one with the best output.

2
Optimize off-chain

A concave-PnL search finds the profit-maximizing injection off-chain — no gas spent searching.

3
Verify & inject

The contract verifies the hint in one step, then the vault deepens the pool for this trade only.

4
Swap & unwind

The trade executes at the better price; the vault withdraws and keeps its share of the fee for LPs.

Roadmap

From proof-of-concept to live infrastructure
✓ Shipped Phase 0 · MVP
Core protocol on OPN testnet
  • Constant-product pools, aggregating router, multi-hop routing
  • JIT vault with on-chain optimizer (concave-PnL ternary search)
  • Deployed & verified on OPN testnet · 11/11 tests passing
● In progress Phase 1 · Hardening
Security & capital efficiency
  • Off-chain compute / on-chain verify — 80% gas reduction
  • Multi-LP vault with pro-rata fee shares + abuse caps
  • Reentrancy guards, safe approvals, permissioned pool registry
→ Next Phase 2 · Robustness
Oracles & MEV resistance
  • TWAP price oracle instead of trusting spot reserves
  • MEV/sandwich protection for the inject-swap-withdraw cycle
  • Split routing — divide a large trade across multiple pools
◇ Later Phase 3 · Mainnet
Production launch
  • Third-party security audit
  • Keeper network for off-chain hint computation
  • Permissionless pool/vault creation, governance, mainnet deploy
